    Carga de Paramétricas¶

    De acuerdo con la tercera regla definida, el proceso de subir la información paramétrica a la base DWH depende de las tablas identificadas en la tabla 2 y que, además, están identificadas con “Sí” en la columna ¿Requiere Carga en dwh?. La herramienta definida para subir los datos fue MySQL Workbench. Sin embargo, durante el desarrollo y las pruebas del modelo analítico, se observó que, por el volumen de información manejado, el uso de MySQL Workbench presentaba tiempos de procesamiento más altos. En este sentido, se plantea el uso de Python como una solución para mejorar la eficiencia. Por ejemplo, para cargar 70 mil registros, MySQL Workbench tomaba cerca de 45 minutos, mientras que con Python, menos de 2 minutos.

    A continuación, se detallan dos maneras de cargar la información a la base de datos DWH para cada fuente:

    Fuente 6: ZoneDimCoor¶

    Esta fuente contiene información de coordenadas para las zonas suministradas por Cajamag. Este origen ya está en DWH con el nombre dwh.BD_Dim_Coordenadas. Se debe actualizar solo si se crea una nueva zona.

    Pasos:

    1. Tomar el archivo Coordenadas.xlsx y agregar la nueva información.
    2. Guardar como CSV.
    3. Ejecutar la ETL 1.9-Dim_Parametricas, la cual se encarga de actualizar DWH con la nueva información.

    Fuente 7: Apo_Proy¶

    Esta fuente contiene información de proyección de aportes. Está en DWH como dwh.BD_Dim_Proyeccion_cumpl_aportes. Se debe actualizar al inicio de cada año.

    Pasos:

    1. Tomar el archivo Proyeccion_cumpl_aportes.xlsx y agregar la nueva información.
    2. Guardar como CSV.
    3. Ejecutar la ETL 1.9-Dim_Parametricas, la cual se encarga de actualizar DWH con la nueva información.

    Fuente 10: MetasCoberturaAfiliacion¶

    Contiene la proyección de afiliación y está en DWH como dwh.BD_Dim_Meta_cobertura_afiliacion. Se recomienda actualizar al inicio de cada año.

    Pasos:

    Detalle de solicitudes de afiliaciones, almacenado en Igestion_SGI, Mercurio_Beneficiarios, Mercurio_Empresas, Mercurio_Trabajadores.

    Pasos:

    1. Tomar los archivos Igestion_SGI.csv, Mercurio_beneficiario.csv, Mercurio_Empresas.csv, Mercurio_Trabajadores.csv y agregar la nueva información.
    2. Guardar como CSV.
