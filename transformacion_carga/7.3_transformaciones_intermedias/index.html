    Transformaciones Intermedias y Satisfacción¶

    Para las fuentes marcadas en la tabla 2 como “Sí” en la columna ¿Requiere Transformación Intermedia?, es necesario aplicar un procesamiento intermedio de transformaciones. A continuación, se detallan los pasos a ejecutar para cada fuente, así como la opción de usar Python para realizar estas transformaciones.

    Fuente 1: Periodos¶

    Esta fuente corresponde a la lista de periodos que se usarán para el análisis en el modelo.

    Pasos:

    1. Ejecutar la ETL 1.3-CalendarioMensual.ipynb, la cual actualiza en DWH la tabla con el nombre dwh.BD_Dim_Calendario.

    Fuente 11: Histórico¶

    Esta fuente contiene información sobre afiliaciones a lo largo del tiempo de empresas, trabajadores y beneficiarios, incluyendo datos sociodemográficos.

    Pasos:

    1. Para la actualización de esta información, ejecutar las siguientes ETL:
    2. 1.6-Fact_Historico_Afiliacion_Empresas.ipynb a DWH con el nombre dwh.BD_Fact_Historico_Afiliacion_Empresas.
    3. 1.7-Fact_Historico_Trabajadores1.ipynb a DWH con los nombres BD_Fact_op_Trabajadores_p1 y BD_Fact_op_Trabajadores_p2.

    4. 1.8-Fact_HistoricoBeneficiarios.ipynb a DWH con el nombre dwh.BD_Fact_Beneficiarios_p2.

    5. Para la actualización de esta información, ejecutar las siguientes ETL adicionales:

    6. 1.6-Fact_Historico_Afiliacion_Empresas.ipynb.
    7. 1.7-Fact_Historico_Trabajadores2.ipynb a DWH con el nombre BD_Fact_op_Trabajadores_p3.

    Fuente 20: Satisfacción¶

    Esta fuente se relaciona con el formato consolidado de encuestas de satisfacción de diferentes servicios y se almacena en la tabla dwh.op_Satisfacción.
